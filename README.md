# GymOS Frontend

Vanilla HTML/CSS/JavaScript frontend для GymOS.

## Запуск

Відкрий `index.html` напряму в браузері.

Застосунок стартує в локальному деморежимі через IndexedDB/localStorage. Режим бекенду можна увімкнути в `Налаштуваннях` після запуску NestJS API у `D:\Features\gymos\gym-os-back`.

## Режим бекенду

Базовий URL API налаштований в `index.html`:

```js
window.FORGE_CONFIG.apiBaseUrl = "http://localhost:3000";
```

У `Налаштуваннях` перемкни `Режим даних` між `Локальний` і `Бекенд API`, потім натисни `Перевірити підключення`.

## AI-тренування (тільки для адміністраторів)

На екрані підготовки тренування адміністратор бачить блок «AI-тренування»: одне текстове поле + кнопка мікрофона. Можна написати або надиктувати тренування, натиснути `Створити за допомогою AI`, перевірити структурований preview і застосувати його — створюється тренування у статусі `Заплановано` (нічого не стартує автоматично).

- **Голос**: диктування виконується нативним мікрофоном клавіатури пристрою (текст іде прямо в поле). Аудіо нікуди не надсилається — у backend іде лише готовий текст. Ліміт довжини: PRO — 2000 символів, admin — 6000 (лічильник символів показується під полем).
- **AI викликається лише через backend** (`POST /ai/workouts/parse`), ключ Gemini у frontend не потрапляє. Блок і вкладка `AI статистика` показуються тільки адмінам; backend додатково перевіряє права (звичайний користувач отримує `403`).
- Налаштування Gemini, змінні середовища, ліміти та статистика описані в README backend-репозиторію.
